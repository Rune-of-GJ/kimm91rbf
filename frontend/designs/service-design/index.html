Pages

홈 대시보드 회원가입 로그인 카테고리 강의 목록 강의 상세 내 강의 강의 시청 진도 관리

P1 서비스 구조

이 시안은 `users`, `categories`, `courses`, `lectures`, `enrollments`, `progresses` 테이블과 현재 API 응답만 기준으로 구성했습니다. KPI, 만족도, 가짜 통계는 제외했습니다.

Users

email, name, role, created_at

Courses

title, description, instructor_name, category

Progress

watched, watched_at, progress_rate

인증

회원가입과 로그인은 `email`, `password`, `name`만 사용합니다. 로그인 유지 방식은 세션입니다.

POST /api/auth/signupPOST /api/auth/login

강의 탐색

카테고리 이름 목록에서 시작해 강의 목록, 강의 상세, 강의 영상 목록으로 이동합니다.

GET /api/categoriesGET /api/courses

학습 진행

수강 신청 후 강의별 시청 여부를 저장하고 코스 단위 진행률을 조회합니다.

POST /api/courses/:id/enrollPOST /api/lectures/:id/progress

현재 시드 기준 예시 데이터

  • 카테고리 5개: 발표 스피치, 면접 스피치, 커뮤니케이션, 설득 화법, 보이스 트레이닝
  • 코스 1개: 스토리텔링 프레젠테이션
  • 강의 3개: 청중 분석과 목표 설정, 메시지 구조화, 오프닝/클로징 설계
  • 내 강의/진도 화면은 이 구조를 기준으로 표시

모든 버튼은 동작하지 않는 디자인 목업이며, 없는 필드는 화면에 추가하지 않았습니다.
