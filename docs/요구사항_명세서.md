# 요구사항 명세서 (P1)

## 개요

P1 단계에서 구현할 기능과 수용 기준(acceptance criteria)을 명세한다.

## 회원가입

- 설명: 사용자는 이메일과 비밀번호를 이용하여 회원가입할 수 있다.
- 수용 기준:
  - 이메일 중복 검사 수행
  - 비밀번호는 안전하게 암호화되어 저장
  - 회원가입 성공 시 사용자 계정 생성 및 생성일 기록

## 로그인

- 설명: 사용자는 이메일과 비밀번호로 로그인할 수 있다.
- 수용 기준:
- 수용 기준:
- 인증 성공 시 액세스 토큰(예: JWT) 또는 세션을 통해 로그인 상태 유지
- 토큰 정책:
  - 액세스 토큰: 짧은 만료(예: 1시간)
  - 리프레시 토큰: 장기 만료(예: 30일) — 필요 시 재발급 흐름 제공
  - 토큰은 안전하게 서명/암호화하거나 충분한 엔트로피(예: 최소 128비트)로 발급
  - 응답에는 `access_token`, `token_type`(Bearer), `expires_in`(초) 포함 권장
- 비밀번호 길이/강도:
  - 최소 길이 8자 권장(복잡도 정책 별도 적용 가능)
- 잘못된 자격증명에 대해 적절한 오류 응답

## 강의 카테고리 조회

- 설명: 사용자는 강의 카테고리 목록을 조회할 수 있다.
- 수용 기준:
  - 카테고리 이름이 목록 형태로 표시

## 강의 목록 조회

- 설명: 사용자는 특정 카테고리의 강의 목록을 조회할 수 있다.
- 표시 항목(수용 기준):
  - 강의 제목
  - 강사 이름
  - 강의 설명

## 강의 상세 조회

- 설명: 사용자는 강의의 상세 정보를 확인할 수 있다.
- 수용 기준:
  - 강의 설명
  - 강의 커리큘럼(강의별 목차)
  - 강의 영상 목록

## 강의 수강 신청

- 설명: 사용자는 강의를 수강 신청할 수 있다.
- 수용 기준:
  - 수강 신청 후 사용자의 내 강의 목록에 추가

## 강의 시청

- 설명: 사용자는 강의 영상을 시청할 수 있다.
- 수용 기준:
  - 강의 영상은 유튜브 링크를 통해 새 창에서 열림(또는 임베드)

## 학습 진도 관리

- 설명: 사용자는 강의 시청 여부를 기록하고 진도를 관리할 수 있다.
- 수용 기준:
  - 특정 강의를 시청 완료로 표시하면 진도 데이터가 저장됨

## 비기능 요구사항

- 보안: 비밀번호는 해시 저장(예: BCrypt). 인증 관련 기본적인 CSRF/세션 보호.
- 확장성: 향후 역할 분리, 결제, 알림 등 기능 추가 용이한 구조
- 데이터 보존: 생성일/수정일 타임스탬프 기록
