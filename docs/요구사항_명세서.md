# 요구사항 명세서 (P1)

## 개요

P1 단계에서 구현할 기능과 수용 기준(acceptance criteria)을 명세한다.
본 문서는 현재 `backend` 구현 기준으로 정리되며, 프론트는 이 문서와 실제 API 응답을 기준으로 화면을 구성한다.

## 회원가입

- 설명: 사용자는 이메일과 비밀번호를 이용하여 회원가입할 수 있다.
- 수용 기준:
  - 이메일 중복 검사 수행
  - 비밀번호는 안전하게 암호화되어 저장
  - 비밀번호 최소 길이 8자 검증
  - 회원가입 성공 시 사용자 계정 생성 및 생성일 기록
  - 회원가입 성공 시 로그인 세션 시작

## 로그인

- 설명: 사용자는 이메일과 비밀번호로 로그인할 수 있다.
- 수용 기준:
  - 인증 성공 시 세션을 통해 로그인 상태 유지
  - 응답에는 `user`, `token_type`, `expires_in` 포함
  - 현재 P1 구현의 `token_type`은 `Session`
  - 잘못된 자격증명에 대해 적절한 오류 응답 반환

## 로그아웃 / 세션 갱신

- 설명: 사용자는 로그아웃할 수 있으며 현재 로그인 상태를 갱신할 수 있다.
- 수용 기준:
  - 로그아웃 시 세션이 만료됨
  - 세션이 유효하면 현재 사용자 정보를 재조회할 수 있음
  - 인증되지 않은 요청은 401을 반환

## 강의 카테고리 조회

- 설명: 사용자는 강의 카테고리 목록을 조회할 수 있다.
- 수용 기준:
  - 카테고리 이름이 목록 형태로 표시
  - 카테고리 상세 조회 시 해당 카테고리의 강의 수 확인 가능

## 강의 목록 조회

- 설명: 사용자는 특정 카테고리의 강의 목록을 조회할 수 있다.
- 표시 항목(수용 기준):
  - 강의 제목
  - 강사 이름
  - 강의 설명
  - 카테고리 이름
  - 강의 수(lectures_count)
  - 현재 로그인 사용자의 수강 여부(enrolled)

## 강의 상세 조회

- 설명: 사용자는 강의의 상세 정보를 확인할 수 있다.
- 수용 기준:
  - 강의 설명
  - 강의 커리큘럼(강의별 목차)
  - 강의 영상 목록
  - 수강 가능 상태(`enrollment_open`)
  - 강의 공개 기간 정보(`start_date`, `end_date`, `enrollment_deadline`, `max_access_days`)

## 강의 수강 신청

- 설명: 사용자는 강의를 수강 신청할 수 있다.
- 수용 기준:
  - 인증된 사용자만 수강 신청 가능
  - 수강 신청 후 사용자의 내 강의 목록에 추가
  - 이미 신청한 강의에 다시 요청해도 중복 생성되지 않음
  - 수강 신청 마감일이 지난 강의는 신청 불가

## 강의 시청

- 설명: 사용자는 강의 영상을 시청할 수 있다.
- 수용 기준:
  - 강의 영상은 YouTube 링크를 사용
  - 강의/강의 목록 응답에는 `video_url`이 포함됨
  - 로그인 사용자는 강의별 시청 여부(`watched`)를 확인할 수 있음

## 학습 진도 관리

- 설명: 사용자는 강의 시청 여부를 기록하고 진도를 관리할 수 있다.
- 수용 기준:
  - 특정 강의를 시청 완료로 표시하면 진도 데이터가 저장됨
  - 수강하지 않은 강의에는 진도 저장 불가
  - 내 강의 목록에서 코스별 진행률 확인 가능
  - 내 진도 조회에서 코스별/강의별 시청 상태를 확인 가능

## 비기능 요구사항

- 보안: 비밀번호는 BCrypt로 해시 저장. 인증 관련 기본적인 세션 보호 적용.
- 확장성: 향후 역할 분리, 결제, 알림 등 기능 추가 용이한 구조
- 데이터 보존: 생성일/수정일 타임스탬프 기록
- 데이터 기준: 프론트는 실제 DB 컬럼 및 계산 가능한 값만 사용하고, DB에 없는 정보를 임의로 구성하지 않음
