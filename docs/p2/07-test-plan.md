# SpeakFlow P2 테스트 플랜

## 문서 목적

이 문서는 `2026-05-07` 기준 P2 구현 범위를 바탕으로 SpeakFlow의 테스트 범위와 우선 확인 항목을 정리한다.

테스트 목적:

1. 역할 분리 기능이 안전하게 동작하는지 확인
2. 멤버십 / 첨삭 흐름이 실제 사용자 입장에서 끊기지 않는지 확인
3. 피드백에서 지적된 리허설/첨삭권 정책이 구조적으로 반영되었는지 확인

---

## 테스트 범위

### 포함

- 로그인 / 회원가입 / 로그아웃
- 관리자 접근 제어
- 관리자 사용자 관리
- 강사 강의 등록 / 강의편 관리
- 학생 멤버십 가입 / 변경 / 해지
- 학생 첨삭권 구매 / 첨삭 요청 / 결과 확인
- 강사 첨삭 완료 처리
- 첨삭권 차감 출처 기록
- 리허설 사용량 계산

### 제외

- 실제 PG 결제 연동
- 이메일 / 알림 발송
- 운영 배포 환경 모니터링
- 대규모 부하 테스트


## 핵심 테스트 시나리오

## A. 공통 / 인증

### A-1. 로그인 성공

- 올바른 계정 정보 입력
- 기대 결과:
  - 세션 생성
  - 메인 또는 대상 페이지 이동

### A-2. 로그인 실패

- 잘못된 비밀번호 입력
- 기대 결과:
  - 로그인 실패 응답
  - 에러 메시지 표시

### A-3. 권한 없는 접근 차단

- 학생이 `/admin` 접근
- 학생이 `/instructor` 접근
- 비로그인 사용자가 멤버십/첨삭 화면 접근
- 기대 결과:
  - 적절히 차단
  - 로그인 또는 거부 페이지로 이동

---

## B. 관리자

### B-1. 최초 관리자 생성

- 관리자 계정이 없을 때 `/admin/setup`
- 기대 결과:
  - 계정 생성 성공
  - 이후 관리자 진입 가능

### B-2. 사용자 관리

- `/admin/users`
- 기대 결과:
  - 목록 조회
  - 역할 변경
  - 삭제 동작

### B-3. 보호 정책

- 마지막 관리자 강등 시도
- 마지막 관리자 삭제 시도
- 현재 로그인한 관리자 자기 자신 삭제 시도
- 기대 결과:
  - 모두 차단

---

## C. 강사

### C-1. 강의 등록

- 제목 / 설명 / 카테고리 / 날짜 / 썸네일 업로드
- 기대 결과:
  - 저장 성공
  - 목록 반영
  - 성공 토스트 표시

### C-2. 강의 등록 검증 실패

- 제목 누락
- 설명 너무 짧음
- 카테고리 누락
- 날짜 역전
- 기대 결과:
  - 저장 실패
  - 검증 메시지 표시

### C-3. 강의편 추가 / 수정

- owned course 기준 추가 / 수정
- 기대 결과:
  - 저장 성공
  - 목록 반영

### C-4. 첨삭 처리

- 큐 진입
- 요청 열기
- 총평 / 타임코드 코멘트 입력
- 기대 결과:
  - 상태 `completed`
  - 학생 화면에 결과 노출

---

## D. 멤버십

### D-1. 플랜 조회

- `/membership`
- `/membership/plans`
- `/membership/checkout`
- 기대 결과:
  - Lite / Pro / Coach 노출

### D-2. 멤버십 가입

- Coach 또는 Pro 가입
- 기대 결과:
  - `subscriptions` 생성
  - 활성 구독 반영
  - Coach면 포함 크레딧 적립

### D-3. 플랜 변경

- Pro -> Coach
- 기대 결과:
  - 기존 구독 `replaced`
  - 새 구독 `active`

### D-4. 멤버십 해지

- `/membership/account`에서 해지
- 기대 결과:
  - `canceled`
  - 활성 구독 없음

---

## E. 첨삭 상품 / 요청

### E-1. 첨삭권 구매

- `/coaching/products`
- 기대 결과:
  - `coaching_purchases` 생성
  - 양수 `coaching_credit_entries` 생성
  - `remaining_credits` 적립

### E-2. 첨삭 요청 생성

- 제목 / 강의 / 강의편 / 파일명 / 메모 입력
- 기대 결과:
  - `feedback_requests` 생성
  - 음수 원장 기록 생성
  - 실제 적립 엔트리 잔량 차감
  - `coaching_credit_usages` 생성

### E-3. 강의 선택 연동

- 먼저 대상 강의 선택
- 기대 결과:
  - 강의편 목록이 해당 강의 기준으로 필터링

### E-4. 첨삭권 부족

- 잔액 0 상태에서 요청 생성
- 기대 결과:
  - 요청 실패
  - 상품 페이지 유도

### E-5. 첨삭권 우선순위

- Coach 포함 크레딧과 구매 크레딧이 함께 있을 때 요청 생성
- 기대 결과:
  - 현재 기본값 `membership_first` 적용
  - 포함 크레딧이 먼저 차감
  - `applied_credit_entry_id`와 `coaching_credit_usages`에 출처 기록

---

## F. 리허설 사용량

### F-1. 월 사용량 계산

- 같은 달에 `rehearsal_submissions` 2건 생성
- 기대 결과:
  - `current_month_rehearsal_usage == 2`

### F-2. 남은 리허설 수 계산

- Pro 플랜 기준 한도 12
- 같은 달 사용량 2
- 기대 결과:
  - `remaining_rehearsal_count == 10`

주의:

- 현재는 계산값까지만 구현됨
- 학생 UI에 잔여 수가 직접 노출되지는 않음

## 비고

- 첨삭권 출처 선택 UI는 아직 없지만, 기본 우선순위와 DB 기록은 검증 대상이다.
