# P2 개발계획서

## 문서 목적

이 문서는 **2026-05-07 현재 SpeakFlow P2의 실제 구현 상태**를 반영한 개발 계획서다.  
초기 P2 목표였던 역할 분리뿐 아니라, 이후 확장된 **월정액 멤버십 + 1:1 첨삭 흐름**까지 현재 범위에 포함한다.

---

## P2 핵심 목표

P2의 핵심은 다음 네 축이다.

1. `student / instructor / admin` 역할 분리
2. 관리자/강사 전용 실제 운영 화면 구축
3. 학생용 월정액 학습 모델 도입
4. SpeakFlow 전용 기능인 `1:1 첨삭 요청 → 강사 피드백` 흐름 구현

---

## 현재 구현 상태

### 완료된 항목

- 역할 기반 접근 제어
  - `student / instructor / admin` 구분
  - 관리자/강사 메뉴 노출 분리
  - 비권한 사용자의 직접 URL 접근 차단
- 인증
  - `회원가입 / 로그인 / 로그아웃 / refresh`
  - 현재 구현은 **세션 기반 인증**
- 관리자 기능
  - 최초 관리자 계정 생성 (`/admin/setup`)
  - 관리자 대시보드 (`/admin`, `/admin/dashboard`)
  - 사용자 목록/검색/역할 변경/삭제 (`/admin/users`)
  - 마지막 관리자 강등/삭제 방지
- 강사 기능
  - 강사 대시보드형 강의 목록 (`/instructor`)
  - 강의 등록 (`/instructor/courses/new`)
  - 썸네일 파일 업로드
  - 강의편 추가/수정
- 학생 학습 기능 확장
  - 월정액 멤버십 소개/비교/가입/해지/플랜 변경
  - 구독 활성 시 강의 시청 가능
  - `내 멤버십`, `내 첨삭 요청` 화면
- 첨삭 기능
  - 첨삭 상품 목록
  - 첨삭권 구매 시 크레딧 지급
  - 첨삭 요청 생성 시 크레딧 차감
  - 강사 첨삭 대기열/검토/완료 처리
  - 학생이 완료된 총평과 타임코드 코멘트 확인 가능
- 공통 UI/UX
  - 하단 토스트 플래시
  - 실제 페이지 적용 전 preview 페이지 흐름 유지

### 진행 중 / 남은 항목

- 관리자 강의 관리 실제화 (`/admin/courses`)
- 관리자 카테고리 관리 실제화
- 강사 첨삭 요청 자동 배정 정책 정리
- 실제 결제(PG) 연동 (P3에서)
- 첨삭 결과 구조 고도화
  - 타임코드 코멘트 개별 구조화
  - 첨삭 파일 업로드/첨부

| 역할 | 설명 | 현재 상태 |
|------|------|-----------|
| student | 강의 탐색, 멤버십 가입, 강의 시청, 첨삭권 구매, 첨삭 요청 | 구현됨 |

| admin | 관리자 계정 생성, 대시보드 확인, 사용자 역할 관리 및 삭제 | 구현됨 |

## 현재 서비스 정책

### 학습 정책

- 기본 강의 시청은 **월정액 멤버십 기반**
- 활성 멤버십이 있으면 강의 상세와 강의 플레이어 접근 가능
- 기존 `Enrollment` 구조는 남아 있지만, 현재 학습 접근 판정은 **멤버십 또는 기존 수강 상태**를 함께 본다

### 첨삭 정책

- `1:1 첨삭`은 기본적으로 **별도 상품**
- `Coach` 플랜은 **월 2회 첨삭 포함**
- 첨삭 요청 1건당 크레딧 1회 차감
- 강사는 요청을 열어 총평과 타임코드 코멘트를 저장한 뒤 완료 처리

### 결제 정책

- 현재 단계는 **모의 결제 / 상태 반영**
- PG 연동은 아직 미구현
- 지금은 구독/구매 액션이 DB에 기록되며, 실제 청구는 일어나지 않음

## 실제 데이터 모델 변화 요약

### 기존 핵심 테이블

- `users`
- `categories`
- `courses`
- `lectures`
- `enrollments`
- `progresses`

### P2에서 실제 추가된 테이블

- `membership_plans`
- `subscriptions`
- `coaching_products`
- `coaching_purchases`
- `coaching_credit_entries`
- `feedback_requests`

### 유지된 설계 포인트

- `courses.instructor_id`로 강사 계정 연결
- `courses.instructor_name`는 화면 호환용으로 아직 유지
- 썸네일은 `thumbnail_url`에 경로 저장
- 첨삭 요청은 `course_id`, `lecture_id`, `instructor_id`, `response_summary`, `response_timecodes`까지 포함

## 기술 스택 (실제 기준)

- Backend: Ruby on Rails 8.1
- Database: PostgreSQL
- Frontend: ERB + Stimulus + Importmap
- Auth: 세션 기반 로그인 + JSON 응답 API
- File Upload: 로컬 파일 저장 (`public/uploads/course_thumbnails`)
- Test: Rails integration test
