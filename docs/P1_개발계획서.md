# P1 개발계획서

## 개발 목표

P1에서는 온라인 스피치 강의 플랫폼의 핵심 학습 기능(MVP)을 구현한다.

### 구현 목표

- 사용자 회원가입 및 로그인 기능
- 강의 카테고리 조회 기능
- 강의 목록 조회 기능
- 강의 수강 신청 기능
- 강의 영상 시청 기능
- 학습 진도 관리 기능

## 개발 기술 스택

- Backend: Ruby on Rails
- Database: PostgreSQL
- Frontend: Rails View (ERB)
- Video: YouTube 링크 기반 강의 영상
- Version Control: Git / GitHub

## 개발 일정 (7주)

- 1주차: 프로젝트 기획 및 요구사항 분석
- 2주차: DB 설계 및 ERD 작성
- 3주차: 회원가입 및 로그인 기능 구현
- 4주차: 강의 목록 조회 기능 구현
- 5주차: 수강 신청 기능 구현
- 6주차: 강의 시청 및 진도 관리 기능 구현
- 7주차: 테스트 및 리팩토링

## 품질 및 배포 고려사항

- 인증 관련 안전한 비밀번호 저장(BCrypt 등)
- DB 마이그레이션 관리(Rails migrations)
- 환경별 설정 관리(개발/스테이징/프로덕션)
- 간단한 시드 데이터로 초기 카테고리 및 샘플 강의 제공

## Rails 사용을 위한 환경 설정 및 초기 작업 (Windows 기준)

1. 개발 환경 설치
   - Ruby 설치: RubyInstaller (권장) 또는 rbenv/uru 사용
   - Node.js 설치: Rails의 자바스크립트 런타임(최신 LTS 권장)
   - Yarn 설치: 프론트엔드 패키지 관리를 위해 권장
   - PostgreSQL 설치 및 서비스 실행

2. 레일즈 앱 생성 (기존 레포에서 새 앱을 만들 경우 해당 경로에서)

```powershell
gem install rails -v 7.0.0
rails new . --database=postgresql
bundle install
```

3. 데이터베이스 설정

- `config/database.yml`에서 PostgreSQL 접속 정보 설정(로컬 유저/비밀번호)
- 데이터베이스 생성 및 마이그레이션

```bash
rails db:create
rails db:migrate
rails db:seed
```

4. 권장 Gem

- `pg` (PostgreSQL)
- `bcrypt` (has_secure_password)
- `dotenv-rails` 또는 `figaro` (환경변수 관리)
- `jbuilder` 또는 `active_model_serializers` (API 응답)
- `kaminari` 또는 `will_paginate` (페이지네이션)

5. 모델/컨트롤러 초기 생성 예시

```bash
rails g model User email:string name:string password_digest:string role:string
rails g model Category name:string description:text
rails g model Course title:string description:text category:references instructor_name:string thumbnail_url:string
rails g model Lecture course:references title:string video_url:string order_no:integer duration:integer
rails g model Enrollment user:references course:references
rails g model Progress user:references lecture:references watched:boolean watched_at:datetime
rails db:migrate
```

6. 간단한 인증 구현 아이디어

- `User` 모델에 `has_secure_password` 사용
- 로그인 세션은 Rails 세션 기반으로 구현하거나 JWT 토큰 사용 가능

7. 시드 데이터 및 테스트 계정

- `db/seeds.rb`에 기본 카테고리와 샘플 코스/강의를 추가

8. 기본 API 라우팅 예시 (`config/routes.rb`)

```ruby
namespace :api do
	namespace :v1 do
		post 'auth/signup', to: 'auth#signup'
		post 'auth/login', to: 'auth#login'

		resources :categories, only: [:index, :show]
		resources :courses, only: [:index, :show] do
			post 'enroll', on: :member
			resources :lectures, only: [:index]
		end

		resources :lectures, only: [:show] do
			post 'progress', on: :member
		end
	end
end
```

9. 권장 개발 흐름

- 브랜치 전략: `main`/`develop`/feature branches
- 커밋 메시지 규칙(간단히): feat/fix/docs 등
- CI: 간단한 테스트 실행(추후 GitHub Actions 권장)
